\begin{abstract}

本文提出一種結合預訓練語言模型（Pre-trained Language Model）、圖神經網路（Graph Neural Network）與對比式學習（Contrastive Learning）之跨架構惡意軟體分析方法。針對不同指令集架構（ISA）下惡意軟體難以統一分析之問題，本研究利用 Ghidra P-Code 中間語言作為架構無關之中介表示，並透過上述模型捕捉語意與結構特徵，以提升跨架構惡意軟體偵測與分類之準確率。實驗結果顯示，本方法在多項基準資料集上優於現有方法。

\textbf{關鍵字：}惡意軟體分析、跨架構、P-Code、預訓練語言模型、圖神經網路、對比式學習
\end{abstract}

% 1. 前言

\section{前言}

物聯網（IoT）生態系統正快速擴張，全球連網裝置數量於 2025 年已達 211 億，預計 2030 年將突破 400 億\cite{sinha2024iot}。這些裝置廣泛運行於 x86、ARM、MIPS 等異質指令集架構（ISA），且往往受限於硬體資源而缺乏完善的安全防護，成為攻擊者鎖定的目標。IoT 裝置遭受的攻擊事件從 2018 年的 3,270 萬次急升至 2022 年的 1.1229 億次\cite{petrosyan2024statista}，資安威脅日益嚴峻。

面對多架構環境，現有惡意軟體分析方法大多針對單一架構設計，無法直接套用至其他平台。由於各架構之特定樣本數量有限，跨架構偵測面臨嚴重的資料不均衡問題。常見的解決方案包括以可印字串（Printable String）\cite{lee2020cross}、中間表示（Intermediate Representation）\cite{phu2019cfdvex,rahat2024cimalir}或屬性控制流程圖（Attributed CFG）\cite{feng2016scalable,yu2023cfg2vec}作為架構無關特徵，並結合對比式學習\cite{redmond2018cross,wang2023retargeted,wang2024learning}或遷移學習等方法，將豐富的常見架構樣本知識遷移至資源稀少之架構。

在中間表示的選擇上，IDA Pro Microcode\cite{rays2021idapro}、Angr VexIR\cite{angr2026}與 Ghidra P-Code\cite{ghidra2023pcode} 均為主流方案。其中，Ghidra 為美國國家安全局（NSA）開發之開源逆向工程工具，其 P-Code 中間語言能將多種架構之機器碼統一轉譯為一致之高階表示，有效縮減架構間的語法差異，為跨架構分析提供共同的語意基礎\cite{naus2022formal}。

本研究以 Ghidra P-Code 作為架構無關之中介表示，提出結合預訓練語言模型（PLM）與圖神經網路（GNN）之跨架構 IoT 惡意軟體偵測框架。預訓練語言模型負責對 P-Code 指令序列進行語意編碼，圖神經網路則建模函式呼叫圖（Call Graph）之結構特徵，兩者協同捕捉跨架構惡意軟體的語意與拓樸資訊，以提升偵測準確率。

\subsection{動機}

現有惡意軟體分析方法大多依賴特定架構之組合語言特徵，無法直接套用至其他架構。此外，不同架構間的語意對齊困難，使得跨架構偵測成為一大研究缺口。本研究動機在於利用架構無關之中間語言與深度學習技術，建立通用性更強之分析模型。

% ============================
% 1.2 貢獻
% ============================
\subsection{貢獻}

本文之主要貢獻如下：
\begin{itemize}
  \item 提出以 Ghidra P-Code 作為跨架構統一表示之分析框架。
  \item 結合預訓練語言模型與圖神經網路，同時捕捉語意與控制流結構特徵。
  \item 引入對比式學習強化跨架構語意對齊能力。
  \item 在多個公開資料集上驗證方法之有效性，並與現有基準方法進行比較。
\end{itemize}

\subsection{惡意軟體分析}

惡意軟體分析方法可分為靜態分析、動態分析與混合分析。靜態分析透過反組譯程式碼抽取特徵，動態分析則透過沙箱執行觀察行為。近年來，深度學習方法被廣泛應用於惡意軟體分類與偵測任務中。

% ============================
% 2.2 跨架構惡意軟體分析
% ============================
\subsection{跨架構惡意軟體分析}

跨架構惡意軟體分析旨在建立能夠處理多種指令集架構之統一模型。現有研究多採用中間表示或遷移學習策略，但仍存在語意對齊不足與泛化能力有限等問題。

% ============================
% 2.3 中間語言 Ghidra P-Code
% ============================
\subsection{中間語言 Ghidra P-Code}

Ghidra 是由美國國家安全局（NSA）開發之開源逆向工程工具，其核心特性之一為將多種架構之機器碼統一轉譯為 P-Code 中間語言。P-Code 具備架構無關性，能有效消除不同 ISA 間的語法差異，為跨架構分析提供一致之語意基礎。

\section{方法}

本節介紹所提方法之整體架構，包含預訓練語言模型、圖神經網路與對比式學習三個主要模組。

% ============================
% 3.1 預訓練語言模型
% ============================
\subsection{預訓練語言模型}

本研究採用預訓練語言模型對 P-Code 指令序列進行語意編碼。透過在大規模程式碼語料庫上預訓練，模型能夠捕捉 P-Code 操作碼（opcode）與運算元（operand）之語意關係，生成豐富之指令嵌入表示。

% ============================
% 3.2 圖神經網路
% ============================
\subsection{圖神經網路}

將函式之控制流程圖（Control Flow Graph, CFG）轉換為圖結構輸入，透過圖神經網路（GNN）聚合鄰居節點資訊，捕捉程式之結構特徵。每個節點代表一個基本塊（Basic Block），節點特徵由預訓練語言模型之輸出提供。

% ============================
% 3.3 對比式學習
% ============================
\subsection{對比式學習}

為強化跨架構語意對齊，本研究引入對比式學習框架。將同一惡意軟體樣本於不同架構下之表示視為正樣本對，不同樣本之表示視為負樣本對，透過對比損失函式縮短正樣本間之距離並拉大負樣本間之距離。

\section{結論}

本文提出一種結合 Ghidra P-Code 中間語言、預訓練語言模型、圖神經網路與對比式學習之跨架構惡意軟體分析框架。透過將不同架構之程式碼統一轉譯為 P-Code，有效消除架構差異；並藉由深度學習模型同時建模語意與結構特徵，提升跨架構偵測能力。未來工作將探索更大規模預訓練模型之應用，以及動態分析特徵之整合。
